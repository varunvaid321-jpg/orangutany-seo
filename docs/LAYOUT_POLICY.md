# Layout Policy

## Principles

- Variation is allowed in presentation, never in factual standards.
- Each species page should feel distinct, not factory-generated.
- Layout patterns are defined in `data/layouts/layout-rules.json`.
- Mood profiles affect visual tone, not validation rules.

## Two-Layer Content Model

1. **Reference layer** (mandatory): taxonomically correct, anatomy-aware images and data
2. **Human layer** (optional): approved contextual images that add personality

## Mood Profiles

- `reference_strict` - pure reference, no contextual imagery
- `field_guide` - field guide feel, forest/habitat context allowed
- `forager_context` - foraging scenes, human holding allowed
- `forest_observation` - nature observation mood
- `culinary_context` - market/kitchen context (edible species only)
- `warning_strict` - toxic/deadly species, no playful imagery

## What May Vary

- Hero aspect treatment
- Gallery position (above or below taxonomy)
- Contextual image placement (early, middle, late)
- Section order for non-critical sections
- Habitat card placement
- Look-alike block placement
- Related species card style
- Whether contextual image appears at all

## What Must Not Vary

- Image approval criteria
- Section-image relevance rules
- Map architecture
- Attribution display
- Disclaimer placement
- Hero validation logic
- Crop safety rules
